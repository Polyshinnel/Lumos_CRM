## Чек-лист тестирования

### Регистрация
- [ ] POST `/api/auth/register` с валидными данными → 201, пользователь создан со статусом `pending`
- [ ] POST `/api/auth/register` с уже занятым email → 422, ошибка валидации
- [ ] POST `/api/auth/register` с коротким паролем → 422, ошибка валидации
- [ ] POST `/api/auth/register` без подтверждения пароля → 422, ошибка валидации

### Авторизация (неподтверждённый пользователь)
- [ ] POST `/api/auth/login` с корректными данными, статус `pending` → 403, «Ваш аккаунт ещё не подтверждён администратором.»

### Ручное подтверждение
- [ ] В БД изменить `status` пользователя на `approved`

### Авторизация (подтверждённый пользователь)
- [ ] POST `/api/auth/login` с корректными данными, статус `approved` → 200, получен JWT-токен
- [ ] POST `/api/auth/login` с неверным паролем → 401, «Неверный email или пароль.»
- [ ] POST `/api/auth/login` с несуществующим email → 401, «Неверный email или пароль.»

### Защищённые маршруты
- [ ] GET `/api/auth/me` с валидным токеном → 200, данные пользователя
- [ ] GET `/api/auth/me` без токена → 401
- [ ] GET `/api/auth/me` с невалидным токеном → 401

### Обновление токена
- [ ] POST `/api/auth/refresh` с валидным токеном → 200, новый JWT-токен

### Выход
- [ ] POST `/api/auth/logout` с валидным токеном → 200, токен инвалидирован
- [ ] Повторный запрос с тем же токеном → 401

### Frontend
- [ ] Страница `/login` — форма отображается, валидация работает
- [ ] Страница `/register` — форма отображается, валидация работает
- [ ] После успешного входа — редирект на `/`
- [ ] Защищённые маршруты недоступны без авторизации → редирект на `/login`
- [ ] После `logout` — редирект на `/login`, токен удалён из `localStorage`
