# Шаг 11. Проверка фиксации header и sidebar при скролле

**Этап:** 4 — Финализация  
**Тип:** Ручная проверка  
**Статус:** [ ] Не выполнен

---

## Цель

Убедиться, что header и sidebar остаются зафиксированными при любом скролле контента.

---

## Тест-кейсы

### 11.1 Header фиксирован

- [ ] При вертикальном скролле контента header остаётся на месте
- [ ] Header занимает полную ширину экрана (поверх sidebar и контента)
- [ ] Высота header — 60px
- [ ] Между header и контентом нет «зазора» или перекрытия

### 11.2 Sidebar фиксирован

- [ ] При вертикальном скролле контента sidebar остаётся на месте
- [ ] Sidebar начинается ниже header (не перекрывает его)
- [ ] Ширина sidebar — 260px
- [ ] Если пунктов навигации больше, чем помещается — sidebar скроллится независимо (через `ScrollArea`)

### 11.3 Контент не перекрывается

- [ ] Контент не заходит под header (есть отступ сверху = высоте header)
- [ ] Контент не заходит под sidebar (есть отступ слева = ширине sidebar)
- [ ] `AppShell` автоматически управляет этими отступами через `padding`

---

## На что обратить внимание

- `AppShell` из Mantine v8 нативно поддерживает фиксацию header и navbar — не нужно добавлять `position: fixed` вручную
- Если элементы «скачут» или перекрываются — проверить правильность конфигурации `AppShell`:

```tsx
<AppShell
    header={{ height: 60 }}
    navbar={{ width: 260, breakpoint: 'sm' }}
    padding="md"
>
```

---

## Критерии завершения

- [ ] Header зафиксирован при скролле контента
- [ ] Sidebar зафиксирован при скролле контента
- [ ] Контент не перекрывается header/sidebar
- [ ] Sidebar скроллится независимо при длинном списке навигации
