# Шаг 10. Проверка скролла основной области

**Этап:** 4 — Финализация  
**Тип:** Ручная проверка  
**Статус:** [ ] Не выполнен

---

## Цель

Убедиться, что основная рабочая область (`AppShell.Main`) корректно скроллится по вертикали и горизонтали, не затрагивая фиксированные header и sidebar.

---

## Тест-кейсы

### 10.1 Вертикальный скролл

**Подготовка:** Временно добавить много контента на страницу (или использовать страницу с большим объёмом данных).

**Ожидаемое поведение:**
- [ ] Контент в `AppShell.Main` скроллится вертикально
- [ ] Header остаётся зафиксированным вверху
- [ ] Sidebar остаётся зафиксированным слева
- [ ] Скроллбар появляется только в области контента, а не на всей странице

### 10.2 Горизонтальный скролл

**Подготовка:** Временно добавить широкий элемент (например, таблицу с множеством колонок или `div` с `min-width: 2000px`).

**Ожидаемое поведение:**
- [ ] Контент скроллится горизонтально
- [ ] Header и sidebar не смещаются при горизонтальном скролле
- [ ] Скроллбар горизонтальный появляется в области контента

### 10.3 Комбинированный скролл

- [ ] Одновременный вертикальный и горизонтальный скролл работает
- [ ] Нет «прыжков» или мерцаний при скролле

---

## Если что-то не работает

### Проблема: скроллится вся страница, а не только контент

Решение — добавить стили на `AppShell.Main`:

```tsx
<AppShell.Main style={{ overflow: 'auto' }}>
```

### Проблема: горизонтальный скролл не работает

Решение — проверить `PageContent` и контейнер контента:

```tsx
<Box style={{ overflow: 'auto' }}>
    {children}
</Box>
```

---

## Критерии завершения

- [ ] Вертикальный скролл работает, header/sidebar фиксированы
- [ ] Горизонтальный скролл работает для широкого контента
- [ ] Комбинированный скролл не вызывает артефактов
- [ ] Временный тестовый контент удалён после проверки
