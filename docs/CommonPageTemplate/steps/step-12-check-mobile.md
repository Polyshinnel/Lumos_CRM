# Шаг 12. Проверка мобильной версии (Burger + Drawer sidebar)

**Этап:** 4 — Финализация  
**Тип:** Ручная проверка  
**Статус:** [ ] Не выполнен

---

## Цель

Убедиться, что на мобильных экранах (ширина < `sm`, т.е. < 768px) layout корректно адаптируется: sidebar скрывается, появляется Burger, sidebar открывается по нажатию.

---

## Как тестировать

- Открыть DevTools браузера → Toggle device toolbar (Ctrl+Shift+M)
- Выбрать мобильное разрешение (например, iPhone 12 — 390×844)
- Или просто уменьшить ширину окна браузера до < 768px

---

## Тест-кейсы

### 12.1 Sidebar скрыт по умолчанию

- [ ] На мобильном экране sidebar не виден
- [ ] Контент занимает всю ширину экрана (без отступа слева под sidebar)

### 12.2 Burger кнопка

- [ ] Burger отображается в header слева (или рядом с логотипом)
- [ ] На desktop (≥ `sm`) Burger скрыт (`hiddenFrom="sm"`)

### 12.3 Открытие sidebar

- [ ] Нажатие на Burger открывает sidebar (как drawer/overlay)
- [ ] Sidebar показывается поверх контента
- [ ] Навигационные пункты отображаются и кликабельны

### 12.4 Закрытие sidebar

- [ ] Повторное нажатие на Burger закрывает sidebar
- [ ] Нажатие на область контента (вне sidebar) закрывает sidebar (если есть overlay)

### 12.5 Адаптация header

- [ ] Имя пользователя скрыто на мобильном (`visibleFrom="sm"`)
- [ ] Аватар/иконка пользователя видна
- [ ] Кнопка выхода видна

### 12.6 Переход между размерами экрана

- [ ] При увеличении ширины > `sm` — sidebar появляется, Burger скрывается
- [ ] При уменьшении ширины < `sm` — sidebar скрывается, Burger появляется
- [ ] Переход плавный, без мерцаний

---

## Если что-то не работает

### Burger не показывается

Проверить, что в `AppHeader`:
```tsx
<Burger opened={burgerOpened} onClick={onBurgerClick} hiddenFrom="sm" />
```

### Sidebar не скрывается на мобильном

Проверить конфигурацию в `AppLayout`:
```tsx
navbar={{ width: 260, breakpoint: 'sm', collapsed: { mobile: !opened } }}
```

### Состояние не синхронизировано

Убедиться, что `opened` и `toggle` из `useDisclosure()` корректно пробрасываются из `AppLayout` в `AppHeader`.

---

## Критерии завершения

- [ ] Sidebar скрыт на мобильном по умолчанию
- [ ] Burger отображается и работает
- [ ] Sidebar открывается/закрывается по Burger
- [ ] Header адаптирован для мобильных
- [ ] Переход desktop ↔ mobile плавный
