# Шаг 1. Подготовка типов — расширение модели User

**Этап:** 1 — Подготовка типов  
**Файл:** `resources/js/features/auth/types/auth.types.ts`  
**Статус:** [x] Выполнен

---

## Цель

Добавить поддержку роли пользователя (`role`) на фронтенде для корректного отображения информации в header (иконка короны для администратора, текст роли).

---

## Проблема

В текущей модели `User` нет поля `role`. Есть только `{ id, name, email, status }`. На бэкенде роль также не реализована — существует только enum `UserStatus`.

---

## Действия

### 1.1 Добавить тип `UserRole`

В файле `auth.types.ts` добавить:

```typescript
export type UserRole = 'user' | 'admin';
```

### 1.2 Расширить интерфейс `User`

Добавить поле `role` в существующий интерфейс:

```typescript
export interface User {
    id: number;
    name: string;
    email: string;
    status: 'pending' | 'approved' | 'rejected';
    role: UserRole;
}
```

---

## Временное решение (fallback)

Поскольку бэкенд пока не отдаёт поле `role` из `/api/auth/me`, во всех местах использования необходимо применять fallback:

```typescript
const userRole = user.role ?? 'user';
```

Это гарантирует, что при отсутствии поля из API пользователь будет считаться обычным пользователем.

---

## Бэкенд (отдельная задача, НЕ входит в текущий шаг)

В будущем необходимо:
- Создать миграцию: добавить поле `role` в таблицу `users`

- Добавить `role` в ответ эндпоинта `/api/auth/me`
